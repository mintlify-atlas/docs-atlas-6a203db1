Copy Fail (CVE-2026-31431) is a Linux kernel vulnerability discovered by [Theori / Xint](https://xint.io) that enables local privilege escalation on a wide range of production Linux distributions. By exploiting a race condition in the kernel's `splice` and AEAD socket subsystems, an unprivileged process can overwrite the contents of otherwise read-only, executable files — such as `/usr/bin/su` — and gain root access.

## What is Copy Fail?

Copy Fail targets a flaw in how the Linux kernel handles cross-VMA (Virtual Memory Area) splice operations combined with AEAD (Authenticated Encryption with Associated Data) sockets. The attack lets an unprivileged attacker write arbitrary bytes into a read-only memory-mapped file by:

1. Opening a privileged binary (e.g. `/usr/bin/su`) without write permissions
2. Creating an AEAD socket with a specially crafted `setsockopt` configuration
3. Using `splice` to transfer data across a pipe into the target file descriptor
4. Repeating the operation in 4-byte chunks to inject a custom payload

The result is a full overwrite of the target binary, which executes as root on the next invocation.
